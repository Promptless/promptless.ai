Promptless now supports Starlight, the Astro-based documentation framework. If your docs live on Starlight, onboarding Promptless no longer requires manual configuration. Promptless detects your Astro config automatically and applies the right defaults.

We've been running our own docs on Starlight for a while, so adding it as a supported platform was overdue.

## The problem

Until now, if you wanted to use Promptless with a Starlight site, you were on your own for configuration. You'd pick "other" in the onboarding flow, point Promptless at your docs repo, and hope the conventions matched. For Astro teams, that meant either skipping Promptless or spending time in setup that should have been automatic.

This matters because Astro (and Starlight on top of it) has been picking up as a docs-site choice. Teams are building new docs on it, and teams with existing docs on other platforms are migrating. Not supporting it meant asking those teams to do work Promptless should have been doing for them.

## How it works

Starlight is now a dropdown option in the onboarding hosting provider list. If you pick it, Promptless applies the right defaults for Astro docs sites.

Usually you won't have to pick it manually. Promptless detects Astro by looking for `astro.config.mjs`, `astro.config.ts`, or `astro.config.js` in your docs repository, and surfaces Starlight as the detected platform automatically.

Promptless also reads the `site` field from your Astro config to figure out your actual docs URL. Earlier versions of onboarding would get confused by "Edit this page" links in Starlight configs and end up pointing at your GitHub repo instead of your actual docs site. That's fixed. If your Astro config has `site: 'https://yourdocs.com'`, or a fallback pattern like `site: process.env.SITE_URL || 'https://yourdocs.com'`, Promptless uses the right value.

## Who benefits most

**Teams shipping new docs on Astro and Starlight.** You can onboard Promptless without learning anything about its configuration conventions. Point it at your docs repo, confirm Starlight in the detected field, and keep going.

**Teams migrating to Starlight from another platform.** The switch doesn't break your Promptless setup. Detection handles the config, and the base URL gets updated automatically when you switch.

**Anyone who was using Promptless with a Starlight site by pretending it was something else.** You can stop doing that now. Re-run onboarding if you want Promptless to pick up the Starlight-specific conventions, or reach out and we can help migrate the config.

## How to use it

If you're onboarding a new docs repo, pick Starlight from the hosting provider dropdown, or let auto-detection do it for you. No additional configuration required.

If you're already onboarded and running a Starlight site, your setup keeps working. You can re-run onboarding on the repo to pick up the improved auto-detection if you want cleaner defaults.

## What's next

We'll keep expanding platform support where it makes sense. If you're using a docs framework that's not yet supported, tell us. The friction should be on us, not you.
