Triggers are events that initiate automated documentation updates in Promptless. When a trigger event occurs, Promptless analyzes the context and determines if documentation updates are needed.

## How Triggers Work

When a trigger event occurs:

1. **Event Detection**: Promptless receives notification of the event (PR opened, message posted, etc.)
2. **Context Analysis**: The system analyzes the event content, including code changes, conversations, or support interactions
3. **Relevance Assessment**: Promptless determines if the event requires documentation updates
4. **Suggestion Creation**: If relevant, Promptless creates documentation suggestions based on the trigger content and connected context sources

## Configuring Triggers

Triggers are configured at the project level in your [projects page](https://app.gopromptless.ai/projects). For each project, you can:

- Select which trigger types to enable
- Configure trigger-specific settings (e.g., repository filters, channel selections)
- Set up auto-publish behavior for documentation updates
- Define notification preferences for your team

<Info>
Need a trigger type that isn't currently supported? Contact us at [help@gopromptless.ai](mailto:help@gopromptless.ai) - we regularly add new trigger types based on user feedback.

## Auto-publish Mode

When auto-publish is enabled for your project, Promptless automatically creates pull requests in your documentation repository with suggested changes. For [commit triggers](/docs/configuring-promptless/triggers/git-hub-commits), you can optionally enable auto-merge to automatically merge documentation PRs as soon as they're created.

Auto-publish is available for all Git-hosted documentation platforms.

## How Promptless Groups Related Triggers

When multiple trigger events affect the same documentation area, Promptless may combine them into a single suggestion rather than creating separate suggestions for each. This reduces noise and keeps related changes together for easier review.

### When Grouping Happens

Promptless groups triggers when:

- Multiple code PRs affect the same documentation pages
- Follow-up messages in a Slack thread add context to an existing suggestion
- A new trigger targets documentation that an open suggestion already covers

### Understanding Grouped Suggestions

When a suggestion includes multiple trigger events, you can see the full list in two places:

- **Dashboard**: Click the suggestion link in any documentation PR description to view all trigger events, citations, and the reasoning process for each change
- **PR description**: The trigger events section lists the sources that informed the documentation updates

The suggestion title and description reflect the combined scope of all grouped triggers. If you see edits that don't seem to match the title, check the trigger events section or open the suggestion in the dashboard to see the full context.

### Viewing Source Citations

Every documentation suggestion includes citations linking specific changes to their sources. To see where a particular edit came from:

1. Open the suggestion in the Promptless dashboard using the link at the top of any documentation PR
2. Review the citations panel, which shows which trigger events informed each section
3. Click any citation to jump to the source—whether it's a GitHub PR, Slack thread, or support conversation

## Trigger Events in Documentation PRs

When Promptless creates a pull request for documentation updates, it automatically includes a list of the trigger events that led to those changes in the PR description. This provides valuable context for reviewers and creates clear traceability between documentation updates and their originating events.
