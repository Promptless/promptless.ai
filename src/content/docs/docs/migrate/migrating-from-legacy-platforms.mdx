Teams often reach Promptless while still on a hosted or legacy platform — Document360, ReadMe, Fern, HubSpot, or RoboHelp — that doesn't expose their docs as files in a Git repository. Moving that content onto a [supported docs-as-code platform](/docs/migrate/choosing-a-platform) is what unblocks adoption. This guide covers the migration pattern every source shares and where the source-specific work lives.

<Info>
A migration is a one-time project, and the effort scales with how much content you have and how far your source format is from Markdown. The export, conversion, and scaffolding steps usually involve someone comfortable with Git and a static site generator, so plan to pair a writer with an engineer if your docs team doesn't write code. No two platforms export the same way, and the Promptless team regularly helps teams move — including offering Starlight migrations — so if you're planning one, reach out at [help@gopromptless.ai](mailto:help@gopromptless.ai) before you start.
</Info>

## The migration pattern

However you export, the shape of the work is the same across source platforms:

<Steps>
  <Step title="Export your existing content">
    Pull your content out of the source platform in the most portable format it offers. Some platforms export Markdown directly; others export HTML or a proprietary format that needs a conversion pass to Markdown or MDX. Capture images and other assets alongside the text so links don't break.
  </Step>

  <Step title="Choose and scaffold a target platform">
    Pick a [supported static site generator](/docs/migrate/choosing-a-platform) and scaffold a new project in a GitHub repository. This repository becomes the source of truth for your docs.
  </Step>

  <Step title="Convert and import the content">
    Move the exported content into the new project's file and folder structure, converting to the target platform's Markdown or MDX conventions. Rebuild the navigation and any frontmatter (titles, slugs, sidebar placement) the new platform expects.
  </Step>

  <Step title="Map old URLs to new ones">
    Record where every existing page lands in the new structure so you can preserve inbound links. See [Preserving URLs and redirects](/docs/migrate/preserving-urls-and-redirects) for how to build and apply that mapping.
  </Step>

  <Step title="Connect the repository to Promptless">
    Once the content builds and the site is in a GitHub repository, connect it as a doc collection. From that point, Promptless keeps it current. See [Doc locations & collections](/docs/connect/doc-locations/github-repos).
  </Step>
</Steps>

## Notes by source platform

The pattern above holds for every source; the differences are mostly in how each platform lets you get your content out.

- **Document360 (Doc360).** A hosted knowledge-base platform. Export your articles and category structure, then convert to your target platform's Markdown or MDX. Recreate the category tree as the new site's navigation.
- **ReadMe.** A hosted API-and-guides platform. Export your guides and reference content and rebuild the navigation in the target platform. Note that ReadMe's newer **Refactored** experience is itself a [supported Promptless platform](/docs/connect/doc-locations/github-repos#supported-platforms) — if your content already lives in a Git-backed ReadMe Refactored project, you may be able to connect it directly instead of migrating away.
- **Fern.** Fern is a docs-and-API platform that Promptless [supports](/docs/connect/doc-locations/github-repos#supported-platforms) when its content is Git-backed. If your Fern content is in a repository, connect it as a doc collection rather than migrating. Only migrate off Fern if your setup can't be reached through a Git repository.
- **HubSpot.** HubSpot stores docs and knowledge-base content in its hosted CMS rather than as repository files. Export the content, convert it from HTML to Markdown or MDX, and rebuild the structure in a Git-backed target platform.
- **RoboHelp.** A legacy authoring tool that publishes to HTML or other help formats. Export to the most portable format available, convert to Markdown or MDX, and reconstruct the topic hierarchy as the new site's navigation.

<Tip>
If your content already lives in a Git repository on a supported platform, you may not need to migrate at all — you can connect it to Promptless directly. Migration is for teams whose docs aren't yet reachable as files in a repository.
</Tip>

## After the migration

With your docs in a GitHub repository on a supported platform:

1. Apply your [URL redirect mapping](/docs/migrate/preserving-urls-and-redirects) so existing links keep resolving.
2. [Connect the repository as a doc collection](/docs/connect/doc-locations/github-repos) and let Promptless finish its [initial analysis of your docs](/docs/connect/doc-locations/how-promptless-learns-your-docs).
3. Configure the [triggers](/docs/connect/triggers) that should start keeping the migrated docs current.

Need a hand with any of this? Contact [help@gopromptless.ai](mailto:help@gopromptless.ai).
