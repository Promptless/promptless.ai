The New Task form lets you request documentation updates directly from the Promptless dashboard. Use it when you want to trigger documentation work without going through Slack, GitHub, or another integration.

## When to use New Task

New Task works well for:

- **Ad-hoc documentation requests** — asking Promptless to document a specific feature, fix outdated content, or add missing information
- **Internal documentation tasks** — requests that don't originate from a support conversation or code change
- **Testing Promptless** — trying out the platform with a simple request before configuring automated triggers

For complex requests that span multiple pages or need extended research, check **Deep Analysis** to enable the [Deep Analysis workflow](/docs/how-to-use-promptless/deep-analysis).

## Submitting a task

<Steps>
  <Step title="Open New Task">
    Click **New Task** in the Promptless sidebar, or go directly to [app.gopromptless.ai/new-task](https://app.gopromptless.ai/new-task).
  </Step>

  <Step title="Write your request">
    Describe what documentation you want Promptless to create or update. Be specific about the topic, the target pages if known, and what changes you want. The first line of your request becomes the task title in the dashboard.

    Good requests include context about what to document and why:

    - "Document the new rate limiting feature. Include configuration options and example responses."
    - "Update the authentication guide to reflect the new OAuth 2.0 flow we shipped last week."
    - "Add troubleshooting steps for the 'connection timeout' error users see during initial setup."
  </Step>

  <Step title="Select a doc collection">
    Choose the doc collection you want Promptless to update. The dropdown shows all doc collections that have finished initial analysis. If you only have one collection, it's selected by default.
  </Step>

  <Step title="Configure delivery settings (optional)">
    Expand **Delivery settings** to customize how Promptless handles the resulting suggestions:

    - **Slack notification channel** — pick a channel to receive progress updates and completion notifications. Leave as **No Slack notification** if you don't need alerts.
    - **Create PRs automatically** — when enabled, Promptless opens a pull request for each suggestion instead of waiting for manual review. Only available for Git-backed doc collections.
  </Step>

  <Step title="Attach files (optional)">
    Click **Add files** or paste directly into the form to attach supporting material like screenshots, PDFs, or specs. You can include up to 5 files, 10 MB each. Attachments give Promptless additional context for your request.
  </Step>

  <Step title="Submit">
    Click **Submit Trigger**. Promptless confirms the submission and shows a link to track progress on the Triggers page.
  </Step>
</Steps>

<Info>
Most tasks complete within a few minutes. You can close the browser tab after submitting — Promptless runs in the background and notifies you through your configured Slack channel when suggestions are ready.
</Info>

## After submission

Once submitted, your task appears on the [Triggers page](https://app.gopromptless.ai/triggers) with a `Manual Task` label. Click the trigger to see:

- **Status** — whether Promptless is still processing, has completed, or encountered an error
- **Suggestions created** — links to any documentation suggestions that resulted from your request
- **Processing details** — what research Promptless did and why it created (or skipped) documentation updates

If Promptless creates suggestions, they appear in your suggestions list for review. From there you can provide feedback, request changes, or open a pull request.

## Tips for better results

- **Be specific** — "Document the webhook retry logic" produces better results than "improve the webhooks page"
- **Include context** — mention the feature name, relevant code paths, or links to source material
- **Scope appropriately** — for large requests that touch multiple sections, use [Deep Analysis](/docs/how-to-use-promptless/deep-analysis) instead of a regular task

Need help with your request? Contact us at [help@gopromptless.ai](mailto:help@gopromptless.ai).
