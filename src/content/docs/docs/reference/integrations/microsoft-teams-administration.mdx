**For tenant administrators**

<Aside type="note">
  The Microsoft Teams integration is in beta. Contact [help@gopromptless.ai](mailto\:help@gopromptless.ai) for more information about enabling this feature for your organization.
</Aside>

This page covers the administrative tasks that let Promptless read the SharePoint-hosted files behind your Teams channels. To install the Promptless Teams app itself, see the [Microsoft Teams integration](/docs/reference/integrations/microsoft-teams) setup guide first.

<Aside type="note">
  These steps require a tenant administrator. If you don't manage your Microsoft 365 tenant, share this page with your SharePoint or Global Administrator.
</Aside>

Once the app is connected, Promptless reads files shared in Teams channels only from the specific SharePoint sites you grant it. You grant that access with a short PowerShell command block that Promptless generates for you in the dashboard and that you run yourself as a tenant administrator. Promptless never creates the grant on its own — it holds an app-only `Sites.Selected` permission and reads only the sites you have explicitly allowed.

## Prerequisites

- The [Microsoft Teams integration](/docs/reference/integrations/microsoft-teams) is installed and connected for your organization.
- You are signed in with a **SharePoint Administrator** or **Global Administrator** role in the tenant.
- You can run PowerShell on your machine. The [Microsoft Graph PowerShell module](https://learn.microsoft.com/en-us/powershell/microsoftgraph/installation) is installed in the first step below if you don't have it yet.

## Grant Promptless access to SharePoint sites

Teams channel files live in SharePoint sites — a team's standard channels share the team's own site, and each private or shared channel has its own. To keep Promptless reading only what you intend, access is scoped per site: you choose the sites Promptless may read, and it can read no others. The grant has three parts — pick the sites in the dashboard, run the generated commands as an administrator, and let Promptless verify the result.

<Steps>
  1. **Install the Microsoft Graph PowerShell module (once per machine).**

     ```powershell
     Install-Module Microsoft.Graph -Scope CurrentUser
     ```

     On Windows PowerShell 5.1, if the install fails on an untrusted repository, trust the PowerShell Gallery first:

     ```powershell
     Set-PSRepository -Name PSGallery -InstallationPolicy Trusted
     ```

     On macOS or Linux, install PowerShell 7 and launch it with `pwsh`:

     ```bash
     brew install --cask powershell
     ```

  2. **Open the SharePoint site picker and select your sites.** On the [Integrations page](https://app.gopromptless.ai/integrations), open the Microsoft Teams integration and launch the SharePoint site picker. Select each site whose channel files Promptless should read. Promptless generates a PowerShell command block for the sites you select, with your tenant ID, the Promptless application ID, and the site IDs already filled in. Sites Promptless can already read appear locked, and the picker generates commands only for newly selected sites.

  3. **Run the generated command block as an administrator.** Copy the block from the picker and run it in PowerShell. It looks like this, with the placeholder values already substituted for you:

     ```powershell
     Connect-MgGraph -TenantId '<tenant-id>' -Scopes 'Sites.FullControl.All'
     $promptlessAppId = '<promptless-app-id>'
     $grant = @{ roles = @('read'); grantedToIdentities = @(@{ application = @{ id = $promptlessAppId; displayName = 'Promptless' } }) }
     $existing = Get-MgSitePermission -SiteId '<site-id>' | Where-Object { ($_.GrantedToIdentities | Where-Object { $_.Application.Id -eq $promptlessAppId }) -or ($_.GrantedToIdentitiesV2 | Where-Object { $_.Application.Id -eq $promptlessAppId }) }
     if ($null -eq $existing) { New-MgSitePermission -SiteId '<site-id>' -BodyParameter $grant }
     ```

     The first line signs you into your tenant. `Connect-MgGraph` opens a browser for interactive sign-in and consent — sign in as the SharePoint Administrator or Global Administrator. `-TenantId` pins the session to that tenant instead of your home tenant, which matters when your admin account is a guest in more than one tenant, and `-Scopes 'Sites.FullControl.All'` is the delegated permission you consent to for this session so the grant can be created.

     The remaining lines run in that authenticated session. For each selected site, they grant the `read` role to the Promptless application, and the `Get-MgSitePermission` check keeps the grant idempotent, so rerunning the block does not create duplicate permissions.

     Before running the grant, you can confirm you are in the right tenant:

     ```powershell
     Get-MgContext
     ```

     Check that the `TenantId` and `Account` fields match the intended tenant and your admin identity. When you finish, disconnect the session:

     ```powershell
     Disconnect-MgGraph
     ```

  4. **Verify access in Promptless.** Back in the site picker, select the option to verify access. Promptless checks each selected site with its own app-only token and records the sites it can now read; any site it still cannot reach is reported as inaccessible. Promptless never writes to SharePoint during this check — it only confirms read access.
</Steps>

<Aside type="tip">
  If you run PowerShell in a headless or SSH session where a browser cannot open, add `-UseDeviceCode` to the `Connect-MgGraph` line for a code-based sign-in flow. Promptless does not add this flag to the generated block, so include it yourself when you need it.
</Aside>

## What Promptless can and cannot read

A grant applies at the SharePoint site level. A team's standard channels all share one site — the team's root SharePoint site — while each private or shared channel has its own separate site. The site picker lists both kinds, so you can grant a team's standard-channel files alongside its private and shared channels, with each private or shared channel appearing as its own selectable site.

When you [connect Microsoft Teams](/docs/reference/integrations/microsoft-teams), Promptless resolves which SharePoint site backs each private or shared channel by reading only the channel's site identity to populate the picker — it never reads the channel's files. This resolution runs as the account that made the connection, and that account sees only the channels it belongs to, so connect as someone who is a member of the private and shared channels you want to cover. That account can differ from the administrator who runs the PowerShell grant. A channel added after you connect becomes grantable only once you reconnect, and any channel whose site Promptless cannot resolve appears in the picker under **Files not grantable yet** rather than as a grantable option.

One case stays outside this model:

- **Personal OneDrive attachments** — files shared from a person's OneDrive rather than a team or channel site — cannot be granted through this flow and remain unavailable.

Promptless reports files on any site you have not granted as unavailable rather than reading them. For how Promptless handles the content it reads, see [Data handling and classification](/docs/security/data-handling-and-classification).

## Removing access

You cannot remove a SharePoint site grant from within Promptless. Microsoft lets a delegated administrator create these grants, but deleting one requires an app-only credential with the `Sites.FullControl.All` application permission — which Promptless does not hold. Already-granted sites therefore appear locked in the site picker rather than offering a revoke action. To remove access, delete the site permission granted to the Promptless application using your own tenant administration tooling that holds that application permission.
