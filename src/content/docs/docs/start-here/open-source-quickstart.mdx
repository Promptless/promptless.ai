Promptless helps open source projects keep documentation in sync with fast-moving codebases. Use it to automatically update developer docs, generate changelogs for new releases, or draft translations.

<Aside type="tip" title="Open Source Program">
  Promptless is free for CNCF, Linux Foundation, and other eligible non-commercial open-source projects. Contact us at [help@gopromptless.ai](mailto\:help@gopromptless.ai) to get started.
</Aside>

Open source workflows sometimes differ from commercial software. We've built features and setup guides tailored specifically for open source maintainers.

## Quick start

The basic setup for your open source project involves three steps:

<section id="promptless-oss-quick-start" class="pl-site-steps is-compact" data-sticky-heading="false">
  <div class="pl-site-step-list">
    <article class="pl-site-step" id="install-the-github-app">
      <span class="pl-site-step-num">1</span>
      <h3>Install the GitHub App</h3>

      <p>
        Connect GitHub from the <a href="https://app.gopromptless.ai/integrations" target="_blank" rel="noopener noreferrer">Integrations page</a>.
        You'll choose which repositories Promptless can access.
      </p>

      <p><strong>Which GitHub app should you install?</strong></p>

      <ul>
        <li>
          <strong>Standard GitHub app</strong>: If you can install apps with write permissions,
          Promptless creates branches and PRs directly on your repo. Works with both public and private repositories.
        </li>

        <li>
          <strong>GitHub (read-only) app</strong>: For public repositories only. If you can only install read-only apps, Promptless creates
          PRs from a fork. Private repositories are not supported with this integration—use the standard GitHub app instead. See the <a href="/docs/reference/integrations/github">GitHub integration docs</a> for
          details.
        </li>
      </ul>
    </article>

    <article class="pl-site-step" id="set-up-doc-collections">
      <span class="pl-site-step-num">2</span>
      <h3>Set Up Doc Collections</h3>

      <p>
        <a href="/docs/connect/doc-locations/github-repos">Doc Collections</a> tell Promptless where your
        documentation lives.
      </p>

      <p>
        You can create multiple doc collections if needed, for example, if your changelog is in a different location
        from your main docs. You can also specify a particular directory if your docs live in a monorepo alongside
        your code.
      </p>

      <p>See <a href="/docs/connect/doc-locations/github-repos">GitHub Repos (Docs as Code)</a> for setup instructions.</p>
    </article>

    <article class="pl-site-step" id="configure-triggers">
      <span class="pl-site-step-num">3</span>
      <h3>Configure Triggers</h3>

      <p>
        Configure triggers in the <a href="https://app.gopromptless.ai/configuration" target="_blank" rel="noopener noreferrer">Configuration page</a> to connect your
        source repos and doc collections.
      </p>

      <p>
        In the <code>triggers</code> section of your <code>promptless.yaml</code>, specify which repos Promptless listens to for PRs.
      </p>

      <Aside type="tip">
        Enable <strong>Replay recent PRs</strong> (for PR triggers) or <strong>Process last 30 days of commits</strong> (for commit triggers) to generate an initial batch of suggestions from your recent history. This helps you calibrate before going live.
      </Aside>

      <p>
        Many open source projects prefer to centralize operations in GitHub so anyone can benefit from Promptless
        without needing an account. If this applies to you, enable <strong>Auto-create PR</strong> so suggestions
        automatically become pull requests. We recommend turning this on after your initial calibration is complete,
        so that you're able to refine Promptless's configuration before rolling it out to all contributors.
      </p>
    </article>
  </div>
</section>

## Recommended trigger configuration

Most projects trigger Promptless on opened PRs. For popular open source projects—where many contributor PRs get closed without merging—consider using [GitHub Commit triggers](/docs/connect/triggers/github-commits) instead. This way, Promptless only runs when PRs are actually merged to your default branch. We recommend changing this setting after your initial calibration.

## Additional use cases

<details open>
  <summary>Translation workflows</summary>

  For translation workflows, you might want the trigger to be the English directory of your docs. Promptless then drafts updates that translate content to the rest of your documentation.
</details>

<details open>
  <summary>GitHub Issues integration</summary>

  If GitHub Issues are part of your workflow, create a GitHub Issues trigger so contributors can tag Promptless in issues to generate doc updates.
</details>

<details open>
  <summary>Updating Contributor PRs (coming soon)</summary>

  For some monorepo use cases, you might want Promptless to add a docs commit to your contributors' existing PRs rather than create separate docs PRs. If you're interested in this feature, contact us at [help@gopromptless.ai](mailto\:help@gopromptless.ai).

  Note: this is not compatible with the read-only version of the Promptless Github app.
</details>
