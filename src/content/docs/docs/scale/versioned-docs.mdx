Teams that maintain more than one version of their documentation keep each version on its own branch. Promptless works with that model at the branch level: point a doc collection at the branch a version lives on, scope a trigger to the branches you care about, and Promptless targets the right base branch automatically when a change comes from a non-default branch.

## Point a collection at a version branch

Open your Promptless configuration, select the doc collection on the **Doc collections** tab, and expand **More filters (optional)**. The **Default branch** setting controls which branch Promptless targets for its pull requests. It defaults to the repository's default branch, so set it to the branch where a version's documentation lives. (In your promptless.yaml this is the `default_branch` field — see the [Configuration Reference](/docs/reference/configuration-reference) for the full doc-collection schema.)

## Scope a trigger to a version branch

On the **Triggers** tab, open a GitHub pull requests trigger and expand **More filters (optional)**. Turn on **Only pull requests targeting these branches** and list the version branches you want, so a change to one version doesn't start documentation work for another. A GitHub commits trigger offers the same choice as **Which branches?** — leave it empty to watch just the default branch. (These map to the per-trigger `branches` field in promptless.yaml, documented in the [Configuration Reference](/docs/reference/configuration-reference#triggers).)

## Let Promptless target the right base branch

There's nothing to configure here — Promptless targets the correct base branch automatically. When a change comes from a non-default branch, Promptless opens the suggestion against that branch rather than the default, which is exactly what you want when a change lands on a version branch. A base branch chip on the suggestion card shows which branch each suggestion targets, and open suggestion branches stay synced with their base branch, so a suggestion always builds on the latest documentation state.

<Aside type="note">
  This page covers versioning at the branch level — the level Promptless models today. Promptless targets branches, not a separate "version" concept, so express each version as the branch its documentation lives on.
</Aside>
