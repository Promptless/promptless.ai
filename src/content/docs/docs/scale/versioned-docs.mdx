Teams that maintain more than one version of their documentation keep each version on its own branch. Promptless works with that model at the branch level: point a doc collection at the branch a version lives on, restrict triggers to the branches you care about, and Promptless targets the right base branch automatically when a change comes from a non-default branch.

## Point a collection at a version branch

Each doc collection has a `default_branch` setting — the branch Promptless targets for its pull requests. It defaults to the repository's default branch, so set it explicitly when a version's documentation lives on a different branch. See the [Configuration Reference](/docs/reference/configuration-reference) for the full doc-collection schema.

## Restrict a trigger to a version branch

When a trigger watches pull or merge requests, its `branches` filter fires the trigger only for requests targeting the branches you list. Use it to scope a trigger to a specific version branch, so changes to one version don't start documentation work for another. GitHub commit triggers take the same `branches` filter to choose which branches they monitor. The Configuration Reference documents the per-trigger `branches` field for [each trigger type](/docs/reference/configuration-reference#triggers).

## Let Promptless target the right base branch

Promptless targets the correct base branch automatically. When a trigger comes from a non-default branch, Promptless creates the suggestion against that branch rather than the default — exactly what you want when a change lands on a version branch. A base branch chip on the suggestion card shows which branch a suggestion targets, and open suggestion branches stay synced with their effective base branch, so a suggestion always builds on the latest documentation state.

<Aside type="note">
This page covers versioning at the branch level — the level Promptless models today. Promptless targets branches, not a separate "version" concept, so express each version as the branch its documentation lives on.
</Aside>
