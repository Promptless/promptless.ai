**Improvements:**

* **Trigger reviews with @promptless in PR descriptions:** Include `@promptless` in a pull request's title or description to trigger a documentation review when the PR opens—even if your project uses "first approval" mode. Useful when you want earlier reviews, when external tools create PRs that need documentation, or when you want to add documentation requests to your PR template. The mention must be a standalone token (not part of an email address). Directory filters still apply; for PRs opened directly against your documentation repository, the mention also bypasses source scope filters.

* **@promptless mentions work on docs repo PRs:** When you open a PR directly against your documentation repository with an `@promptless` mention, Promptless now triggers a review even if that repository is not in your project's source scope. Docs repositories are typically excluded from source triggers during onboarding—the explicit mention acts as an opt-in.

**Bug Fixes:**

* **Suggestion Diff Editor Markdown Rendering:** Fixed bold formatting bleeding into subsequent lines when inline code appeared alongside bold text in the suggestion diff editor—markdown now renders correctly regardless of formatting combinations.
