**Improvements:**

* **Trigger reviews with @promptless in PR descriptions:** Include `@promptless` in a pull request's title or description to trigger a documentation review when the PR opens—even if your project uses "first approval" mode. Useful when you want earlier reviews, when external tools create PRs that need documentation, or when you want to add documentation requests to your PR template. The mention must be a standalone token (not part of an email address), and repository scope and directory filters still apply.
