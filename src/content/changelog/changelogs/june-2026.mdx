**Improvements:**

* **Smarter Tab Badge Counts:** Status tab badges (Ready, PR Open, Closed) now respect your active filters. Previously, filtering by docs location or other criteria showed unfiltered totals in the badges—now they update to reflect only matching suggestions.

* **Multi-Status Tab Highlighting:** When your query selects multiple statuses—like `(is:ready OR is:pr_open)`—all matching tabs are highlighted. Adjacent selections merge into a single highlight bar for a cleaner look.

* **Clear Filters and Sort:** A "Clear filters and sort" button now appears below the query input whenever you've changed filters or sort order from the defaults. One click resets to `is:ready`, sorted by newest, with no other filters applied.

* **Sort Control Relocated:** The "Sort by" dropdown moved from the filters row to the tab row, opposite the status tabs for quicker access.

* **Search by Docs PR Number:** The `pr:` search qualifier now lets you find suggestions by their docs PR number—type `pr:123` or `pr:#123` to locate the suggestion that opened PR #123. Suggestions with an open docs PR also display a clickable badge linking directly to the PR.

**Bug Fixes:**

* **Suggestion Diff Editor Markdown Rendering:** Fixed bold formatting bleeding into subsequent lines when inline code appeared alongside bold text in the suggestion diff editor—markdown now renders correctly regardless of formatting combinations.
