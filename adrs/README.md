# Architecture Decision Records

This directory holds **Architecture Decision Records (ADRs)** — short documents that capture a
significant decision, the context that forced it, the options considered, and the chosen outcome.
They are the durable record of *why* this site is set up the way it is.

We use the [MADR](https://adr.github.io/madr/) (Markdown Any Decision Records) format, adopted from
the Promptless [Starport template](https://github.com/Promptless/starport-template) this site was
migrated onto.

## When to write an ADR

- **Any architecture change.** Switching hosts, adding a major plugin, changing the docs framework,
  restructuring navigation, or re-syncing to a newer Starport template release.

Small, reversible content edits do **not** need an ADR.

## How to add one

1. Copy [`0000-madr-template.md`](0000-madr-template.md) to `NNNN-short-kebab-title.md`, using the
   next number in sequence.
2. Fill in the sections. Keep it short — a screenful is plenty.
3. Set the status (`proposed` → `accepted`). Don't edit accepted ADRs to reverse them; write a new
   ADR that supersedes the old one and update the old one's status to `superseded by NNNN`.
